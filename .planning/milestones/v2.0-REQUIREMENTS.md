# Requirements: RedWardenLite Evolution

**Defined:** 2026-02-26
**Core Value:** Legitimate traffic must pass reliably while non-conformant or suspicious traffic is blocked predictably with auditable policy reasons.

## v2.0 Requirements

### Extensibility

- [x] **EXT-01**: Formal plugin capability/version compatibility metadata is defined and enforced at load-time boundaries.
- [x] **EXT-02**: Plugin execution isolation and boundary controls are strengthened with deterministic failure semantics.

### Distributed Operations

- [x] **DIST-01**: Multi-node policy state distribution and coordination model is implemented with deterministic convergence behavior.
- [x] **DIST-02**: Centralized fleet-level telemetry aggregation and retention controls are implemented with operational runbook guidance.

## Future Requirements

### Advanced Platform Operations

- **CTRL-01**: Optional control-plane UI/API for fleet policy and runtime profile management.
- **AUTO-01**: Automated rollback orchestration driven by policy/transport/observability health thresholds.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Runtime control-plane web UI in v2.0 | Requires separate product surface and auth model; deferred to future requirements |
| Full protocol expansion beyond HTTP/HTTPS proxy in v2.0 | Current value and test contracts are centered on HTTP/HTTPS behavior |
| Autonomous self-healing rollback actions | Deferred until distributed telemetry + control-plane assumptions are validated |
